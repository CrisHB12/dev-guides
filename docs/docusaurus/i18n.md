# Internacionalización (i18n)

Docusaurus tiene un excelente soporte nativo para **i18n** (internacionalización), lo que te permite traducir tu sitio web a múltiples idiomas fácilmente.

De forma predeterminada, los sitios de Docusaurus están en inglés, pero puedes configurar otros idiomas.

## 1. Configuración

Para habilitar múltiples idiomas, abre tu archivo `docusaurus.config.ts` y busca la sección `i18n`.

Ejemplo para habilitar **Inglés** (por defecto) y **Español**:

```typescript title="docusaurus.config.ts"
const config: Config = {
  // ...
  i18n: {
    defaultLocale: 'en', // Idioma por defecto
    locales: ['en', 'es'], // Lista de idiomas soportados
    localeConfigs: {
      en: {
        label: 'English',
      },
      es: {
        label: 'Español',
      },
    },
  },
  // ...
};
```

## 2. Traducción de Contenido

El sistema de traducción de Docusaurus se basa en tener carpetas separadas para cada idioma dentro del directorio `i18n/`.

### Estructura de Directorios

Cuando configures i18n, tu estructura de archivos crecerá para incluir:

```text
website/
├── docs/                   # Documentación en el idioma por defecto (ej. Inglés)
├── i18n/
│   ├── es/                 # Traducciones al Español
│   │   ├── docusaurus-plugin-content-docs/
│   │   │   └── current/    # Aquí van tus archivos .md traducidos
│   │   ├── docusaurus-theme-classic/
│   │   │   └── navbar.json # Traducciones del tema (navbar, footer)
│   │   └── code.json       # Traducciones generales
```

### Traduciendo Documentos Markdown

1.  Copia los archivos de `docs/` a la carpeta de traducción correspondiente.
    *   Ejemplo: Copia `docs/intro.md` a `i18n/es/docusaurus-plugin-content-docs/current/intro.md`.
2.  Traduce el contenido del archivo copiado al español.
3.  **Mantén el mismo `id`** en el Front Matter (si está definido) para que Docusaurus sepa que es la misma página.

### Traduciendo el UI (Barra de navegación, Footer)

Para traducir elementos de la interfaz como la barra de navegación o el pie de página, utiliza el comando:

```bash
bun run write-translations --locale es
```

Esto generará archivos JSON en `i18n/es/`. Abre estos archivos y traduce los valores de las cadenas de texto.

Ejemplo de `navbar.json`:

```json
{
  "title": {
    "message": "Mi Sitio",
    "description": "The title in the navbar"
  },
  "item.label.Tutorial": {
    "message": "Tutorial",
    "description": "Navbar item with label Tutorial"
  }
}
```

## 3. Visualización Local

Para ver tu sitio en un idioma específico mientras desarrollas, usa la bandera `--locale`:

```bash
bun run start --locale es
```

Esto iniciará el servidor de desarrollo mostrando la versión en español de tu sitio.

## 4. Despliegue

Al ejecutar `bun run build`, Docusaurus generará automáticamente la versión estática para **todos** los idiomas configurados.

Normalmente:
- El idioma por defecto estará en la raíz (`/`).
- Los otros idiomas estarán en subdirectorios (`/es/`, `/fr/`, etc.).

:::tip Selector de Idioma
Recuerda agregar un selector de idioma en tu barra de navegación (`navbar`) en `docusaurus.config.ts`:

```typescript
navbar: {
  items: [
    // ...
    {
      type: 'localeDropdown',
      position: 'right',
    },
  ],
},
```
:::
